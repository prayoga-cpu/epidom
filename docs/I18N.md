# Internationalization (i18n)

## Overview

EPIDOM supports multiple languages using a custom i18n implementation with React Context.

## Supported Languages

| Code | Language   | Coverage |
| ---- | ---------- | -------- |
| `en` | English    | 100%     |
| `fr` | French     | 100%     |
| `id` | Indonesian | 100%     |

---

## File Structure

```
src/locales/
├── en.ts      # English translations (~118KB)
├── fr.ts      # French translations (~127KB)
├── id.ts      # Indonesian translations (~115KB)
└── index.ts   # Export aggregator
```

---

## Usage

### In React Components

```typescript
import { useI18n } from '@/components/lang/i18n-provider';

function MyComponent() {
  const { t, locale, setLocale } = useI18n();

  return (
    <div>
      <h1>{t('dashboard.title')}</h1>
      <p>{t('common.welcome')}</p>

      <select value={locale} onChange={(e) => setLocale(e.target.value)}>
        <option value="en">English</option>
        <option value="fr">Français</option>
        <option value="id">Bahasa Indonesia</option>
      </select>
    </div>
  );
}
```

### Translation Keys

Translations are nested objects:

```typescript
// locales/en.ts
export const en = {
  common: {
    welcome: "Welcome",
    save: "Save",
    cancel: "Cancel",
    delete: "Delete",
  },
  dashboard: {
    title: "Dashboard",
    overview: "Overview",
  },
  data: {
    materials: {
      title: "Materials",
      addButton: "Add Material",
      deleteConfirm: {
        title: "Delete Material",
        description: "Are you sure you want to delete {name}?",
      },
    },
  },
};
```

### Variable Interpolation

Use `{variable}` syntax:

```typescript
// Definition
"deleteConfirm.description": "Are you sure you want to delete {name}?"

// Usage
t('deleteConfirm.description').replace('{name}', material.name)
```

## Adding a New Language

1. Create new locale file:

```typescript
// src/locales/de.ts
export const de = {
  common: {
    welcome: "Willkommen",
    // ... copy structure from en.ts
  },
};
```

2. Register in index:

```typescript
// src/locales/index.ts
import { de } from "./de";

export const locales = { en, fr, id, de };
export type Locale = keyof typeof locales;
```

3. Update config:

```typescript
// src/config/app.config.ts
export const locales = ["en", "fr", "id", "de"] as const;
```

## User Locale Preference

Locale preference is stored:

1. In user's browser (localStorage)
2. In user profile (database)

```typescript
// Saved to user profile on change
async function updateLocale(userId: string, locale: string) {
  await prisma.user.update({
    where: { id: userId },
    data: { locale },
  });
}
```

## Date & Currency Formatting

Locale-aware formatting:

```typescript
import { useCurrency } from '@/components/providers/currency-provider';

function PriceDisplay({ amount }) {
  const { formatPrice, currency } = useCurrency();

  return <span>{formatPrice(amount)}</span>; // €10.00 or $10.00
}
```

## Best Practices

1. **Never hardcode strings** - Always use `t('key')`
2. **Use semantic keys** - `common.save` not `button1`
3. **Keep translations synchronized** - All locales must have same keys
4. **Context-specific keys** - Use feature namespaces like `data.materials.title`
